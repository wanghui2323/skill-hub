# Skill 安全评估六维评分标准

## 评分总览

满分 100 分，由以下六个维度组成：

1. 来源可信度（25 分）
2. 社区热度（20 分）
3. 任务明确性（20 分）
4. 可复用性（15 分）
5. 安全性（10 分）
6. 跨平台兼容（10 分）

---

## ① 来源可信度（满分 25 分）

**评分依据**：

- **25 分**：来自 Anthropic 官方仓库（anthropics/*, anthropic/* 组织下）
- **20 分**：知名 AI 工具维护者，仓库 Star > 1,000
- **15 分**：活跃社区贡献者，仓库 Star 500-1,000
- **10 分**：普通社区用户，仓库 Star 100-500
- **5 分**：新建账号或仓库 Star < 100
- **0 分**：匿名账号、账号创建不足 30 天、无法验证来源

**判断方法**：
- 检查 GitHub URL 中的组织名/用户名
- 检查仓库的 Star 数量（从 README 或 URL 判断）
- 如果是本地文件，标注"无法验证来源"，给 5 分

---

## ② 社区热度（满分 20 分）

**评分依据**：

- **Star 数量**（10 分）
  - 10 分：Star > 500
  - 7 分：Star 200-500
  - 5 分：Star 50-200
  - 3 分：Star 10-50
  - 0 分：Star < 10

- **活跃度**（5 分）
  - 5 分：近 6 个月有代码提交
  - 3 分：近 1 年有代码提交
  - 0 分：超过 1 年无更新

- **社区认可**（5 分）
  - 5 分：出现在主流 Awesome 合集或官方推荐
  - 3 分：有完整文档和使用示例
  - 0 分：无文档或示例

**判断方法**：
- 从 URL 或 README 推断 Star 数
- 查看 SKILL.md 更新日期（如有 Git 信息）
- 检查是否有详细的使用说明

---

## ③ 任务明确性（满分 20 分）

**评分依据**：

- **description 清晰度**（10 分）
  - 10 分：一句话说清功能，无歧义
  - 7 分：功能明确但稍冗长
  - 4 分：描述模糊但可理解
  - 0 分：包含"让 Claude 更智能"等空洞描述

- **触发词精确性**（10 分）
  - 10 分：有明确的使用场景示例
  - 7 分：描述了使用时机但不够具体
  - 4 分：只有功能描述，无触发场景
  - 0 分：完全没有说明何时使用

**判断方法**：
- 阅读 YAML frontmatter 中的 `description` 字段
- 检查 SKILL.md 中是否有 "When to use" 或使用示例
- 评估是否能从描述中判断何时触发

---

## ④ 可复用性（满分 15 分）

**评分依据**：

- **路径通用性**（8 分）
  - 8 分：无硬编码路径，使用相对路径或环境变量
  - 5 分：有少量硬编码但不影响核心功能
  - 2 分：多处硬编码路径
  - 0 分：完全依赖特定路径结构

- **环境依赖**（7 分）
  - 7 分：无特殊环境依赖，纯文本处理
  - 5 分：依赖常见工具（git, npm, python）
  - 3 分：依赖特定项目结构或配置
  - 0 分：只能在特定环境运行

**判断方法**：
- 搜索 SKILL.md 和 scripts 中的路径（如 `/home/`, `/Users/`, `C:\`）
- 检查是否要求特定的项目结构
- 评估换一个电脑能否正常使用

**常见问题**：
- ❌ `/home/username/projects/`
- ❌ 假设必须有 `src/` 目录
- ✅ 使用 `$HOME` 或 `~`
- ✅ 通过 Read 工具动态查找文件

---

## ⑤ 安全性（满分 10 分）

**评分依据**：

- **网络请求**（5 分）
  - 5 分：无网络请求
  - 3 分：只请求官方 API（如 GitHub, npm registry）
  - 0 分：有任意 URL 请求或外部数据获取

- **权限最小化**（5 分）
  - 5 分：只使用 Read/Grep/Glob 等只读工具
  - 3 分：使用 Write/Edit 但有明确说明
  - 1 分：使用 Bash 但无危险命令
  - 0 分：使用 Bash 执行网络命令或系统修改

**判断方法**：
- 检查 scripts 中是否有 `curl`, `wget`, `fetch`, `requests`
- 查看 SKILL.md 中提到的工具列表
- 搜索 `http://`, `https://` 出现的位置

**危险信号**：
- ❌ `curl http://` 下载外部内容
- ❌ `rm -rf` 删除文件
- ❌ `eval` 执行任意代码
- ⚠️ `fetch()` 访问 API（需判断是否可信）

---

## ⑥ 跨平台兼容（满分 10 分）

**评分依据**：

- **路径分隔符**（5 分）
  - 5 分：使用 POSIX 标准路径（`/`）或路径库
  - 3 分：混用 `/` 和 `\` 但有兼容处理
  - 0 分：硬编码 Windows 路径（`C:\`）

- **平台 API**（5 分）
  - 5 分：不依赖特定平台 API
  - 3 分：依赖常见跨平台工具（git, node）
  - 0 分：调用平台专属命令（如 Windows 的 `dir`）

**判断方法**：
- 检查是否用 `os.path.join()` 或 `pathlib`（Python）
- 查找 Windows 特定命令（`dir`, `cls`, `type`）
- 评估在 Claude.ai 网页端和 Claude Code 中是否都能用

**兼容性示例**：
- ✅ `path/to/file.txt`
- ✅ Python `pathlib.Path()`
- ❌ `C:\Users\file.txt`
- ❌ Bash 的 `source ~/.bashrc`

---

## 总分判断标准

- **≥ 70 分**：✅ 建议安装
  - 输出完整的复制安装指引
  - 给出具体的安装命令和路径

- **50-69 分**：⚠️ 谨慎安装
  - 列出具体风险点（如"有硬编码路径"）
  - 说明可能遇到的问题
  - 由用户自行决定是否安装

- **< 50 分**：❌ 不建议安装
  - 指出主要失分维度
  - 说明为何存在风险
  - 建议寻找替代方案
